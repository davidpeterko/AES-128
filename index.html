AES 128

Who was it for?

This was a project for the course Intro to Computer Security

What was it?

It is an encryption algorithm based on AES 256, but instead is just a little inferior and is the AES 128 version, but has the same principles of diffusion and confusion.

When was it?

This was done in the month of June 2015.

Where was this?

This was done at the University of Texas computer lab.

Why was it done?

Test my Java programming skills and knowledge of cryptography to try and understand the ins and outs of the AES.

How was it done?

It was done in Java using the structure of how AES 256 was implemented: using the functions subBytes, shiftRows, mixColumns, and addRoundkey. I read alot on Rijndael to figure out the subtle differences between AES 128, 192, and 256. This required a good understand of key expansion, to which I used alot of internet references as a supplement.
